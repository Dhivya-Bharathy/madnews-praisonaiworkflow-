Thin Line - Daily Politics Workflow

Architecture for scraping top 10 daily political stories and showing LEFT / CENTER / RIGHT analysis on click.

1) End-to-End Flow

A. Daily Scheduler

Runs once every morning.
  • Trigger cron job
  • Collect candidate stories
→

B. Ingestion + Dedup

NewsAPI / RSS / source feeds.
  • Normalize titles/URLs
  • Cluster same story variants
→

C. Perspective Engine

Generate LEFT/CENTER/RIGHT points.
  • Source-grounded bullet extraction
  • Blind-spot generation
→

D. Store Daily Snapshot

Persist top stories + analyses.
  • Story table
  • Perspective table
→

E. UI Consumption

User clicks story card.
  • Fetch precomputed analysis
  • Render links under each side

2) Proposed System Architecture

Backend Services

  • Scheduler Service: daily trigger for top political stories.
  • Ingestion Service: fetches, normalizes, and clusters stories.
  • Classification Service: tags as politics / policy / elections.
  • Analysis Service: builds LEFT/CENTER/RIGHT + blind spots.
  • API Service: exposes daily list + story detail endpoints.

Data Layer

  • stories_daily (top 10 list per day)
  • story_sources (raw source links, outlet, timestamp)
  • story_perspectives (LEFT/CENTER/RIGHT bullets)
  • story_blindspots (left/center/right ignores)
  • analysis_audit (model version, run time, quality flags)

3) Click Workflow (User Experience)

  • User lands on “Today’s Top 10 Politics Stories”.
  • User clicks one story card.
  • API returns precomputed perspective payload in <1 second (cache-first).
  • UI shows three columns: LEFT, CENTER, RIGHT.
  • Each bullet includes Source Link Section under that side.
